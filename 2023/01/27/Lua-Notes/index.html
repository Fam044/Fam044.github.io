标识符与关键字

  • Lua的标识符以字母与下划线开头，数字不能开头，不允许使用特殊字符

  • Lua区分大小写的

  • 注意：最好避免用“下划线+大写字母”，因为系统Lua使用（比如：_Age 不推荐使用）

  • Lua中的关键字：
    and break do else elseif
    end false for function if
    in local nil not or
    repeat return then true until
    while

  • 推荐命名规则

    • 常量使用全大写和下划线：MY_ACCOUNT
    • 变量的第一个字母小写： strNumber
    • 全局变量第一个字母用小写g表示： gMyAccount
    • 函数名第一个字母大写： funtion MyFirstFunction()

变量

变量不需要在使用前说明，且不需要指定变量的类型

  • 变量都是弱类型，类似JavaScript，无需指定变量类型
  • Lua语句是否以分号结尾都可以正常运行
  • Lua中单双引号都可以使用
  • 变量名不能以数字开头
  • 推荐使用类似C#命名规范
  • C#必须先定义再使用，而Lua可以直接使用变量
  • Lua变量可以随时改变类型
  • 使用type()方法获取变量类型
  • 变量的类型
    • nil：表示空数据，等同于null
    • boolean：布尔型，Lua中除了nil和false，其余都表示“真”
    • string：字符串类型，可以使用单双引号
    • number：小数类型，Lua中没有整数类型
    • table：表示集合，序号从1开始
    • function：表示由Lua编写的函数
    • userdata：表示任意存储在变量中的C数据类型
    • thread：表示执行的独立线程（本质是“伪多线程”，是协程的概念），用于执行协同程序

多行注释： –[[注释内容]]–

Lua数据类型与运算符

字符串常用基础

  • 字符串可用单引号和双引号
  • 可以使用两个方括号，来表示 “一块”字符串（多行字符串）
  • 字符串拼接使用”..”
  • 如果字符串使用”+”号，会自动将字符串转换为number类型进行处理（必须保证该字符串为 数字字符串，否则报错）
  • 字符串与数值类型，使用”..”拼接一般不会报错；但是数值型可能为nil，建议使用tostring函数

运算符

  • 算数运算符

    + - * / % ^

    Lua中没有自增和自减运算符

  • 关系运算符

    < <= > >= == ~=

    nil只与自身相等

    对于table、函数、userdata，Lua是做引用比较的。即：只有当他们引用同一个对象时，才认为他们相等

  • 逻辑运算符

    and or not

    所有逻辑运算符将false, nil视为假，其他视为真

    • 有短路逻辑
      • 对于操作符and，如果第1个操作数为假，就返回第1个操作数，不然返回第2个操作数
      • 对于操作符or，如果第1个操作数为真，就返回第1个操作数，不然返回第2个操作数

多重赋值

  • Lua中可以给多个变量同时赋值
  • Lua可以进行多重赋值
    • Lua总是将等号右边值的个数调整到与左边变量个数相一致
    • 如果值的个数少于变量个数，多余变量就会被赋值为nil
    • 如果值的个数多余变量个数，多余变量就会被忽略

全局与局部变量

  • Lua变量都是全局性的，定义局部变量需要使用关键字local
  • Lua语句块（if, for等）与函数中，定义的默认也都是全局变量
  • Lua可以直接使用do…end来直接定义一个语句块
  • 要删除一个全局变量，可以给变量赋值为nil

ipairs与pairs迭代器

  • for循环分为 数值循环，泛型循环（相当于foreach）
  • ipairs遍历顺序的集合（中间序号不能中断，遇到nil数值就停止），一般用于数组类型集合的遍历
  • pairs遍历集合中所有数据（序号可以中断），一般用于键值对类型集合的遍历
  • 泛型循环定义：
  • 使用pairs关键字时，循环输出“数组”类型的表时，for中必须使用k, v；使用pairs关键字，循环中的k, v字母，可以是任何字母或者合法标识符代替
  • Lua标准库中提供了几种迭代器
    • 迭代文件按每行的io.lines
    • 迭代table元素的pairs
    • 迭代数组元素的ipairs
    • 迭代字符串中单词的string.gmatch等

break关键字

函数

定义函数

  • 函数的定义格式如下

  • Lua函数的基本性质

    • 函数无需定义返回类型，可以返回任意类型和任意数量的数值
    • 函数的参数，无需定义参数类型
    • 函数无需大括号
    • 可以定义变量，将函数直接赋值给变量，从而获得相同功能
    • 注意：函数中由于我们函数的参数没有类型，所以在没有（编译）运行前，计算机是无法发现错误的

全局与局部变量

  • Lua中定义的变量默认是全局变量，且在函数内部定义的变量也默认是全局变量
  • 函数可以声明为全局的与局部的，默认是全局的

函数的多返回值

  • 一个函数返回多个数值，可以使用多个变量直接接受
    • 如果一个函数调用在最后，或者仅有一个表达式，则Lua会保留尽可以多的返回值，用于匹配
    • 如果一个函数没有返回值，或者没有足够多的返回值，那么Lua会用nil来补充缺失数值

函数的赋值

函数可以作为数据进行赋值，也可以作为参数进行传递（相当于C#中的委托）

匿名函数

  • 匿名函数，即没有名称的函数
  • 调用方法：一个没有名称的函数，只有通过赋值给一个变量，通过调用这个变量来间接调用这个匿名函数

  • 由数字、字母、下划线组成的一串字符
  • Lua语言中字符串可以使用以下三种方式来表示：

    • 双引号间的一串字符
    • [[]]间的一串字符

转义字符串

  • 常用转义字符串
    • 回车： \r
    • 换行： \n
    • 反斜杠： \\
    • 双引号： “”

字符串常用函数

  • 字符串长度：len()
  • 大小写：upper(), lower()
  • 查找：find()
  • 截取函数：sub()
  • 字符串替换函数：gsub()

表

表（Table）的定义

  • Table是Lua的一种数据结构，用来创建不同的数据类型，如：数组、键值对集合等
  • Lua的下表可以是负数
  • Lua的长度可以动态改变
  • 可以把Table当成一个字典集合来对待，也可以当成一个“数组”来对待，要看如何初始化Table
  • Table，可以有“空表”、“直接声明且定义表内容”、“声明然后逐一赋值”三种方式
  • Table也可以定义成类似“数组”的定义方式，可以使用下标来访问（注意：Lua中没有数组的概念，就是用表来表示）
  • 访问Table中数据（键值对类型），可以直接使用”.”来访问，也可以使用中括号访问，但是使用中括号一定要使用字符串形式，否则会报错
  • Table中的下标，从1开始
  • Table中的数值修改方式：
    • 直接移除数组中数据，索引不会变化
    • 使用table专用删除函数，索引会自动排序（推荐使用）

使用迭代器函数输出

  • ipairs函数
    • 输出连续数值
  • pairs函数
    • 输出连续与非连续（数值对）数值
  • Lua中的table相当于C#中的字典类集合（具有键值对的数据结构）
  • 表的遍历方式
    • table.getn()
      • 表示集合中数量（集合中最大的索引）

表函数

  • table.getn()
    • 得到表（数组类型）的长度
    • 注意：该函数不能正确输出 键值对 类型表的长度（’#’同理）
  • table.concat()
    • 表的链接
    • 将表中数据进行链接后进行输出
    • 可以指定分隔符，以及输出的字符串序号
  • table.insert()
    • 在相应位置中插入数值
    • 1表示开头，以此类推
  • table.remove()
    • 表中数据的移除
    • 默认移除最后一位
  • table.sort()
    • 表的排序
    • 对于英文字符串类型，按照字母表排序，先大写后小写
  • table.maxn()
    • 获取表中最大数值，但系统函数有bug（输出了表的长度），需自定义
  • 表资源的释放
    • 使用nil对表（字符串、其他基本数据类型）资源进行释放
    • 给整个table赋值nil，则表示销毁整个表（清空内存）

Lua中的OOP编程

  • Lua的语法没有面向对象的机制
  • Lua中使用Table实现OOP，把Table模拟成一个“类”来使用
  • 其实Lua中的string(math, random)等可以看成面向对象的特殊表

定义字段和方法

  • 使用”tableName.field”，定义表的字段
  • 使用”tableName.methodName()”，定义表的方法
  • 可以将匿名方法赋值给表字段，来定义表

表对象self关键字的作用

  • 方法中直接引用表字段
  • 方法中直接引用表字段的缺陷
  • 定义局部表引用变量，提高方法引用表字段的耦合性
  • 使用self关键字，直接在方法中引用表自身字段（定义方法时要使用冒号”:”）
