数组

数组的基础理论：

  • 数组是存放在连续内存空间上的相同类型数据的集合
    • 所以数组元素是不可以删除的，只可以覆盖
    • 所以增删数组元素时，难免要移动其他元素
    • 如果要释放，只能整体释放
  • 数组的下标都是从0开始的
    • 操作时注意元素的下标

二分查找

给定一个 n 个元素有序的（升序）整型数组 nums 和一个目标值 target ，写一个函数搜索 nums 中的 target，如果目标值存在返回下标，否则返回 -1。

提示：

  • 你可以假设 nums 中的所有元素是不重复的。
  • n 将在 [1, 10000]之间。
  • nums 的每个元素都将在 [-9999, 9999]之间。

前提：

  • 数组为有序数组
  • 数组中无重复元素
    • 若有重复元素，则返回的下表不唯一

思路：

  • 二分法
    • 注意边界，二分法经常写乱就是因为边界的定义不清晰
    • 通常边界的定义分两种：左闭右闭，或 左闭右开

反思：

  • 思想很简单，但是实现起来并不一定可以得心应手，要注意循环不变量原则，把握好区间的定义

移除元素

给你一个数组 nums 和一个值 val，你需要 原地 移除所有数值等于 val 的元素，并返回移除后数组的新长度。

不要使用额外的数组空间，你必须仅使用 O(1) 额外空间并 原地 修改输入数组。

元素的顺序可以改变。你不需要考虑数组中超出新长度后面的元素。

思路：

  • 暴力法
    • 两层for循环，一层for循环遍历数组元素，一层for循环更新数组
    • 遇到待删除元素时，除了需要将size–，还需要将外层循环的指针i–
      • 因为下标i以后的数值都往前移动了1位，所以i也要向前移动1位
  • 双指针法（快慢指针法）
    • 快指针：寻找新数组的元素，新数组就是不含目标元素的数组
    • 慢指针：指向更新 新数组下表的位置

反思：

  • 通过双指针的方法可以将时间复杂度，从O(n^2)降至O(n)，在处理类似的数组或链表问题时，多考虑能否使用双指针法

有序数组的平方

给你一个按 非递减顺序 排序的整数数组 nums，返回 每个数字的平方 组成的新数组，要求也按 非递减顺序 排序。

提示：

  • 1 <= nums.length <= 10^4
  • -10^4 <= nums[i] <= 10^4
  • nums 已按 非递减顺序 排序

思路：

  • 暴力法
    • 将每个数平方后，对数组进行排序（快排）
  • 双指针法
    • 负数平方后会变为正数，而原有正数已经有序，可以考虑从最小正数为起点，建立两个指针，分别向左右两侧移动
    • 也可以从两侧往中间（较简洁）
    • 可以看成左右两个数组进行归并排序，不过比较时稍微修改以下条件

长度最小的子数组

给定一个含有 n 个正整数的数组和一个正整数 target 。

找出该数组中满足其总和大于等于 target 的长度最小的 连续
子数组 [numsl, numsl+1, …, numsr-1, numsr] ，并返回其长度。如果不存在符合条件的子数组，返回 0 。

提示

  • 1 <= target <= 10^9
  • 1 <= nums.length <= 10^5
  • 1 <= nums[i] <= 10^5

思路：

  • 暴力法
    • 利用两层for循环，遍历所有子序列，并记录其中最小的元素个数
  • 滑动窗口法
    • 也可以理解为双指针法，但是题目要求的是两指针间的元素之和是否大于目标值
    • 一个指针指向滑动窗口的终止位置，一个指针指向滑动窗口的起始位置；只使用1个for循环进行遍历，遍历时使用指向终止位置的指针为判断条件；当窗口内的值大于或等于目标值后，固定终止指针，将起始指针逐步往后移，缩小窗口大小直到不满足条件为止，多次循环，便可遍历所有子序列

反思：

  • 通过滑动窗口的方法，不断动态地调整窗口的方法，以寻找满足题设的最优解，并将时间复杂度由O(n^2)降至O(n)

螺旋矩阵Ⅱ

给你一个正整数 n ，生成一个包含 1 到 n2 所有元素，且元素按顺时针顺序螺旋排列的 n x n 正方形矩阵 matrix 。

提示：

  • 1 <= n <= 20

思路：

  • 该题对算法的考察并不多，主要在于模拟过程的能力
  • 该题要注意坚持 循环不变量原则，坚持每条边左闭右开的原则
  • 模拟顺时针画矩阵的过程，由外向内一圈一圈这么画下去
    • 填充上行从左到右
    • 填充右列从上到下
    • 填充下行从右到左
    • 填充左列从下到上
